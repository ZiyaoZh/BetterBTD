# 自动任务指南

BetterBTD 的自动任务不是简单的“循环运行一个脚本”。它会先识别当前游戏界面，再决定怎么导航、什么时候进关、什么时候启动脚本、什么时候处理结算界面。

对用户来说，自动任务页的核心是两件事：

- 给任务准备好可用脚本
- 把脚本通过脚本 ID 绑定到正确任务槽位

## 使用自动任务前必须满足的前提

- 截图器能稳定抓到游戏画面
- 游戏热键和 BetterBTD 按键绑定一致
- 你至少已经验证过一个单脚本可以稳定运行
- 对应任务需要的脚本已经导入到“我的脚本”

如果这四条里有一条没满足，自动任务大概率会失败，而且排查成本会比单脚本高很多。

## 自动任务和脚本库是什么关系

自动任务并不直接操作外部任意文件，而是优先依赖托管脚本库。

真正参与自动任务的是这些东西：

- 托管脚本
- 脚本 ID
- 任务绑定文件
- 订阅包

换句话说，自动任务最适合配合“我的脚本”页一起用，而不是绕开脚本库各自为战。

## 脚本绑定文件是什么

对于收集、金气球、黑框这三类任务，页面上的“配置脚本”按钮会打开一个文本配置文件。它的用途很直接：

- 某个任务槽位应该使用哪个脚本 ID

你可以把它理解成“任务地图槽位”和“脚本库里的脚本”之间的映射表。

这几个文件通常位于：

- `User\MyScripts\Bindings\collection.json`
- `User\MyScripts\Bindings\goldballoon.json`
- `User\MyScripts\Bindings\blackborder.json`

## 什么时候该用绑定文件，什么时候直接填脚本 ID

- 收集、金气球、黑框
  用绑定文件
- 循环刷关
  直接填单个脚本 ID
- 奥德赛
  直接按关卡顺序填 3 到 5 个脚本 ID

## 自动任务页的导入按钮有什么区别

导入按钮有两种入口：

- 导入订阅包
  适合整套收集、金气球、黑框资源迁移。它会同时带脚本和绑定关系。
- 导入单个脚本
  适合把一个脚本迅速塞进脚本库，并顺手用于循环刷关这类任务。

如果你拿到的是别人整理好的整包资源，优先用订阅包；如果你只是新写了一个 `.btd`，就用单脚本导入。

## 通用设置怎么理解

每个任务卡片都有“操作间隔”。

它控制的是自动任务在导航、点击、按键这类动作上的节奏。一般来说：

- 数值更小，动作更快，但环境不稳时更容易误判
- 数值更大，动作更保守，速度会慢一些

如果你不是在专门调优速度，保守一点通常更稳。

## 收集任务怎么配

收集任务适合做活动收集循环。

它的特点是：

- 先识别当前活动地图
- 再从对应“收集模式 + 地图”的绑定槽位里找到脚本
- 进入关卡后运行对应脚本

页面里你需要做的是：

1. 选择收集变体
2. 打开脚本配置文件
3. 给对应地图槽位填入脚本 ID
4. 启动任务

当前支持的收集变体包括：

- 普通收集
- 双金收集
- 快速路径收集
- 双金快速路径收集

实际该选哪个，取决于你的脚本是按哪套前提写出来的。

## 金气球任务怎么配

金气球任务的用法和收集任务类似，但绑定范围更聚焦在新手地图。

推荐流程：

1. 先确认你有能处理对应新手图的脚本
2. 打开金气球绑定文件
3. 给目标地图填入脚本 ID
4. 启动任务并观察识图是否正确

如果你发现任务总是找不到脚本，优先检查的不是标签，而是该地图槽位是否真的绑定了脚本 ID。

## 黑框任务怎么配

黑框任务是目前自动任务里最“队列化”的模式。

你可以：

- 选择地图分组
- 选择单张地图或整组地图
- 让程序按地图、难度、模式逐项推进

它的脚本绑定粒度也最细，因为黑框要区分：

- 地图
- 难度
- 模式

所以一张图可能需要多份脚本来覆盖完整黑框流程。

比较实用的配置顺序是：

1. 先只给一张地图配脚本
2. 先跑通几个模式
3. 再扩展到整张图
4. 最后再扩展到整组地图

不要一开始就把整个分类全开，否则你很难判断是哪一关缺脚本。

## 黑框订阅导出怎么理解

黑框任务支持按不同范围导出订阅：

- 新手图
- 中级图
- 高级图
- 专家图
- 单张地图

这很适合做分批分享、分阶段备份和按图包迁移。

## 循环刷关怎么配

循环刷关最简单，因为它不需要绑定文件。

你只需要：

1. 在脚本库里找到目标脚本
2. 复制脚本 ID
3. 粘贴到“循环刷关”的脚本 ID 输入框
4. 启动任务

这个模式适合：

- 反复刷同一张图
- 验证单个脚本长时间稳定性
- 在进入更复杂自动任务前先做压力测试

## 奥德赛怎么配

奥德赛任务需要你按关卡顺序填 3 到 5 个脚本 ID。

规则很明确：

- 前 3 个必填
- 最多可填到 5 个
- 中间不能跳空

最重要的限制是：

- 当前版本不会从主菜单完整进入奥德赛流程
- 你需要先手动进入奥德赛相关界面，再让 BetterBTD 接管后续阶段

所以奥德赛的推荐用法是“半自动起步，后续自动推进”，而不是完全从桌面一键跑到底。

## 自动任务运行时窗口怎么看

每次启动任务都会打开一个运行时窗口。

你可以在里面看到：

- 当前任务阶段
- 当前识别到的 UI 状态
- 导航日志
- 当前正在运行的脚本
- 当前脚本执行到哪一步
- 连续导航失败次数

这比只看游戏画面更适合排错。

## F10 在自动任务里的作用

和脚本运行窗口一样，自动任务运行窗口会尝试注册全局 `F10` 热键。

- 未运行时按 `F10`，等价于开始
- 运行中按 `F10`，等价于停止

如果 F10 注册失败，通常是系统里有别的软件占用了它。

## 订阅包适合什么时候用

当你满足下面任一条件时，订阅包都比“单文件手动搬运”更合适：

- 你要迁移整套收集脚本
- 你要分享一套金气球配置
- 你要备份一整批黑框绑定和脚本
- 你想在多台机器之间同步任务配置

因为订阅包带走的不只是脚本文件，还有“脚本和任务槽位之间的关系”。

## 实际使用中的推荐顺序

1. 先用单脚本验证截图和输入
2. 再导入脚本库
3. 再配置绑定文件或脚本 ID
4. 再跑单一任务模式
5. 最后再导入整包订阅或扩展地图范围

这是最不容易把问题混在一起的顺序。

## 常见失败原因

### 任务一开始就提示找不到脚本

- 对应绑定文件没填
- 填的是错误脚本 ID
- 目标脚本已经被删掉
- 你以为标签会自动匹配脚本，但当前任务真正依赖的是绑定或脚本 ID

### 进入了关卡，但脚本没启动

- 自动任务没能在进关前正确解析脚本
- 运行时窗口里可以看“当前脚本”是否为空
- 先回到脚本库单独运行那份脚本确认它本身可用

### 黑框模式总在某些地图停住

- 该地图某个难度或模式没有绑定脚本
- 脚本元数据和实际目标不一致
- 地图范围一下开太大，先缩小到单张图排查

### 奥德赛模式直接失败

- 你还停留在主菜单，没有手动进入奥德赛流程
- 你填的脚本 ID 数量不足 3 个
- 脚本 ID 中间有空位

## 最后给用户的一个实用判断标准

如果你在自动任务里遇到问题，先不要立刻怀疑识图。

优先按这个顺序排查：

1. 脚本本身能不能单独跑
2. 脚本 ID 或绑定文件有没有填对
3. 游戏热键是否一致
4. 截图是否正常
5. 当前任务模式是否满足它自己的前提条件

按这个顺序查，通常比一开始盯着识图和日志高效得多。
