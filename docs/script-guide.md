# 脚本与脚本库指南

这份文档讲的是 BetterBTD 里“脚本从哪里来、怎么整理、怎么编辑、怎么运行”。

## 先理解三个概念

- 托管脚本库
  指 BetterBTD 维护的脚本仓库。你导入的脚本、导出的脚本、脚本 ID、自动任务绑定关系都围绕它展开。
- 脚本 ID
  每个托管脚本都有唯一 ID。自动任务、绑定文件、订阅包都依赖这个 ID 定位脚本。
- 脚本元数据
  包括地图、难度、模式、英雄、说明和标签。它决定脚本怎么被筛选、展示和理解。

## “我的脚本”页能做什么

“我的脚本”页是脚本库入口，适合做日常管理。

- 导入脚本
- 导出脚本
- 删除脚本
- 复制脚本 ID
- 按地图、难度、模式、关键字筛选
- 查看脚本说明、英雄、标签和状态
- 直接跳到编辑器
- 直接运行脚本

## 支持导入哪些文件

脚本库页面主要支持这些来源：

- BetterBTD 当前脚本格式：`.btd`
- 旧脚本格式：`.btd6`
- 旧版脚本包：`.btd6s`
- 当前格式的 `.json` 脚本文件

对用户来说最重要的判断标准不是扩展名，而是“这个文件是不是 BetterBTD 能理解的脚本”。如果不确定，优先保留为 `.btd`。

## 导入后的实际行为

导入并不是单纯把文件原样堆进目录里。BetterBTD 会把它纳入托管脚本库，记录：

- 显示名称
- 脚本 ID
- 来源文件名
- 地图、难度、模式、英雄
- 标签
- 更新时间

如果你重复导入相同脚本，程序会尽量把它当作更新而不是制造重复垃圾条目。

## 脚本状态怎么看

脚本库里的脚本可能出现这些状态含义：

- 就绪
  文件和元数据都正常，可以编辑、运行和绑定到自动任务。
- 文件缺失
  库记录还在，但实际脚本文件不在了。
- 元数据异常
  文件还在，但关键信息不完整或不符合当前读取规则。

看到异常状态时，不建议直接继续绑定自动任务，先处理脚本本身。

## 推荐的脚本整理方式

建议把脚本命名和标签都整理好，不要只靠文件名凑合。

比较实用的做法是：

- 名称写清地图、打法或用途
- 说明里写明适用前提和风险点
- 元数据里的地图、难度、模式、英雄保持准确
- 标签用于组织和筛选，而不是代替脚本绑定

要特别注意一点：自动任务当前真正依赖的是“绑定文件或脚本 ID”，不是只靠标签自动猜脚本。

## 脚本编辑器的工作流

脚本编辑器适合从零搭建脚本，也适合修改导入脚本。

一个典型流程是：

1. 新建脚本或从脚本库打开现有脚本。
2. 先填元数据，包括地图、难度、模式、英雄、说明、标签。
3. 从左侧指令库逐条拖入或添加到序列。
4. 给每条指令设置目标、防御塔、数量、坐标或等待条件。
5. 保存后运行，观察运行时窗口和日志。
6. 根据结果回到编辑器微调。

## 编辑器里有哪些指令

当前脚本编辑器提供的主要指令类型包括：

- 放置防御塔或英雄
- 升级指定目标
- 切换目标优先级
- 设置选中塔的能力
- 卖出目标
- 使用英雄物品
- 触发技能栏技能
- 鼠标点击
- 快进或发送下一轮
- 等待时间、金钱、回合或某个坐标颜色
- 修改已记录的塔坐标
- 注释

其中“注释”不会直接操作游戏，只用于说明脚本意图。

## 坐标应该怎么取

BetterBTD 提供了更适合用户的取点方式，不建议手打坐标。

推荐流程：

1. 先确保截图器已经启动。
2. 在编辑器中选中那条需要坐标的指令。
3. 点击“选择坐标”。
4. 把鼠标移动到游戏窗口目标位置。
5. 右键确认。

程序会把脚本坐标、窗口坐标和当前像素颜色一起显示出来。对于“等待颜色”这类指令，这个流程尤其有价值。

## 关于坐标精度你需要知道什么

- 脚本坐标是按 1920×1080 参考系记录的
- 程序会按当前窗口尺寸做换算
- 非 16:9 比例时，换算可能有偏移

所以最稳妥的做法仍然是用 16:9 游戏窗口写脚本、调脚本、跑脚本。

## 指令里的“目标对象”是什么意思

当脚本里有多只同类猴子时，程序不能只靠“这是一只忍者猴”来区分目标。编辑器会自动维护每次放置产生的对象顺序和关联关系，后续升级、卖出、切换目标之类的操作都依赖这层关联。

对普通用户来说，只需要记住两点：

- 放置顺序会影响后续可选目标
- 删除或复制放置指令后，最好再检查一遍后续目标是否还指向正确对象

## 高级参数什么时候需要改

很多指令带有“高级”区域，例如：

- 放置检测
- 放置失败后的微调重试
- 升级检测
- 猴子面板检测
- 卖出检测
- 点击间隔
- 下一条指令前的等待时间

默认值更适合大多数脚本。只有在你已经确认主流程正确，但个别操作节奏不稳时，再去改这些参数。

## 保存、另存为与托管脚本库

推荐标准保存格式是 `.btd`。

如果你是从脚本库打开脚本再保存，BetterBTD 会同步更新托管脚本库中的对应记录。如果你执行“另存为”到不同路径，它更像是在生成一个新脚本副本，你需要再确认它在脚本库里的归属和名字是否符合预期。

## 怎么运行脚本

你有两种常见入口：

- 在“我的脚本”页选中脚本后直接运行
- 在脚本编辑器里运行当前工作区

第二种方式的优势是你可以在保存前先跑当前草稿。

## 运行时窗口怎么看

脚本运行时窗口会显示：

- 指令序列
- 当前执行到哪一步
- 每一步的状态
- 运行日志
- 间隔策略

你可以选择两种节奏模式：

- 按每条指令自己的间隔运行
- 用统一的全局操作间隔覆盖大多数操作

这非常适合做排错实验。

## F10 的作用

脚本运行时窗口打开后，会尝试注册全局 `F10` 热键。

- 未运行时按 `F10`，等价于开始
- 运行中按 `F10`，等价于停止

如果系统里已有别的软件占用了 `F10`，注册可能失败，这时直接点窗口里的按钮即可。

## 一套实用的写脚本顺序

建议按下面节奏写，而不是一次写满所有步骤：

1. 先只写开局放塔
2. 再补前几次升级
3. 再补等待条件
4. 最后再加卖出、技能、下一轮等复杂动作

每加一段就跑一次，这样最容易定位问题。

## 旧脚本迁移时要注意什么

BetterBTD 能读取旧版 `.btd6` 和 `.btd6s`，但迁移不等于百分之百无差别复现。

迁移后建议你做这些检查：

- 地图、难度、模式、英雄是否正确
- 目标对象是否还对应正确的猴子
- 带坐标的能力或点击是否仍然合理
- 注释和备注里是否提示了迁移风险

旧包脚本导入完成后，最好逐个做一次试跑。
